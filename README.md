# toy_ufdb

Union-Find / DSUを利用したDBを自作できないか試している。

## バージョンについて

メジャーバージョンごとにリポジトリを分けていて、このリポジトリはバージョン0。

|メジャーバージョン|内容|
|---|---|
|0|コア機能。オンメモリのみ（プログラム終了でデータは消える）。HTML出力機能|
|1|ストレージ実装（予定）|
|2|TCP接続対応（検討中、着手の予定なし）|

永続化なし・値なし（キーのみ管理）など、v0のスコープの詳細は [`docs/ROADMAP.md`](docs/ROADMAP.md) の「既知の制約」を参照。

内部実装・計算量は [`docs/architecture.md`](docs/architecture.md)、Union-Find自体の一般論は [`docs/union-find.md`](docs/union-find.md) を参照。

## 何ができるの？

1. 最小単位である要素（以下、**キー**）を登録する
2. キー同士をつなげて**集合**を作る。また、集合同士をつなげることもできる
3. 任意の2つのキーが同じ集合に属するかを判定する

6つのキー、`a` / `b` / `c` / `d` / `e` / `f`を追加した様子。これらはつながっていないため、それぞれが別の集合に属している

![何ができるのかを示すイメージ図](docs/images/what-it-does.png)

1. `INSERT` でキーを2つ登録する（まだ互いに無関係）
2. `SAME` で判定すると `false`（まだ同じ集合ではない）
3. `MERGE` で2つのキーを統合する
4. 同じ `SAME` を呼ぶと、今度は `true` が返る（同じ集合になった）

## 使い方

`cargo run` するだけで起動する。

SQLに相当する、このDBの命令文の呼び名を **UFQL**（Union-Find Query Language）とする。REPL上で1行ずつ入力する。

| コマンド | 役割 |
| --- | --- |
|`INSERT <key>`|新しいキーを1つ登録する |
|`MERGE <a> <b>`|キー`a`とキー`b` が属する集合を1つに統合する（未登録キーは自動で登録される） |
| `SAME <a> <b>` | `a` と `b` が同じ集合に属するかを判定する。属するなら `true`、属しないなら `false` が返る |
|`GROUPS` | 代表元ごとに集合の中身をまとめて一覧表示する |

`exit` / `quit` でREPLを終了する。`help` は clap が自動生成するため個別実装は不要。

## 用語集

|用語|意味|
|---|---|
|UFQL|Union-Find Query Language。SQLに相当する、このDBへの命令文の呼び名|
|キー|INSERT/MERGEなどでユーザーが指定する文字列。DBが管理する要素の識別子|
|集合|互いにMERGEされたキーのまとまり|
|グループ|GROUPSコマンドが表示する、集合ごとの一覧単位|
|DB|名前付きのUfdbインスタンス|
